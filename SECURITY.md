# Security policy

## Supported version

Security fixes are applied to the latest code on the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's **Report a vulnerability**
button in the repository Security tab to create a private security advisory.

Include the affected endpoint or file, reproduction steps, expected impact, and any suggested
mitigation. You should receive an initial response within seven days. Please allow time for a fix
before publishing details.

MoveMind binds to `127.0.0.1` and is designed for local use. Exposing the server to an untrusted
network is outside the supported deployment model.
